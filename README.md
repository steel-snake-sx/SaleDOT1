# SaleDOT

SaleDOT - дипломный проект 2022 года: настольная система учета для небольшой точки продаж. Приложение помогает вести товары, клиентов, поставщиков, продажи, закупки, кассовые операции и базовую финансовую отчетность.

Проект написан как WPF-приложение для Windows на C# и .NET Framework 4.8. Для доступа к данным используются MySQL, Dapper и Entity Framework.

## Возможности

- Авторизация пользователей и разделение ролей: `superadmin`, `admin`, `user`, `customer`, `vendor`.
- Главная панель с краткой статистикой по продажам, клиентам, поставщикам и персоналу.
- Справочник товаров: добавление, редактирование, поиск, штрих-коды, цены, скидки и остатки.
- POS-окно для оформления продаж с режимом поиска по штрих-коду.
- Оформление продаж и закупок, просмотр деталей операций.
- Учет клиентов, поставщиков и сотрудников.
- Финансовый блок: счета, балансы, транзакции, расходы и закрытие кассы.
- Журнал складских операций и отчет по стоимости остатков.
- Настройки подключения к базе данных и базовые настройки печати чеков.
- Экспорт/печать документов через Excel-шаблоны и ReportViewer.

## Структура проекта

```text
SaleDot/
  SaleDot.sln                  # Visual Studio solution
  SaleDot/                     # основной WPF-проект
    bll/                       # бизнес-логика и вспомогательные сервисы
    data/                      # модели, репозитории, настройки подключения
    Views/                     # окна приложения
      finance/                 # продажи, закупки, касса, счета, транзакции
      product/                 # товары и складские отчеты
      user/                    # пользователи, клиенты, поставщики
      others/                  # настройки, SQL-запросы, SMS, аккаунт
      report/                  # просмотр отчетов
    assets/                    # шаблоны и графические ресурсы
```

В репозитории также остались резервные каталоги Visual Studio за 2022 год (`SaleDot_Backup_*`, `MigrationBackup`). Они сохранены как часть исходного состояния проекта, но для запуска актуальная версия находится в `SaleDot/SaleDot`.

## Запуск

1. Склонировать репозиторий:

```bash
git clone https://github.com/steel-snake-sx/SaleDOT1.git
cd SaleDOT1
```

2. Открыть решение:

```text
SaleDot/SaleDot.sln
```

3. В Visual Studio восстановить NuGet-пакеты и собрать проект в конфигурации `Debug|x86` или `Release|x86`.

4. Подготовить MySQL и указать параметры подключения в окне настроек базы данных приложения. По умолчанию используются значения:

```text
Server: localhost
Database: saledot
User: root
Password: 1234
```

5. Запустить приложение из Visual Studio.

## База данных

Приложение рассчитано на MySQL. При старте оно проверяет подключение к серверу и наличие базы данных. В коде предусмотрено создание базы из файла `data/saledot.sql`.

Важно: в текущем состоянии репозитория файл `SaleDot/SaleDot/data/saledot.sql` отсутствует. Для полноценного запуска может понадобиться восстановить SQL-дамп базы данных или создать схему вручную по моделям и репозиториям из каталога `SaleDot/SaleDot/data`.

Локальные параметры подключения хранятся в `SaleDot/SaleDot/data/databaseconnection.json` и настройках приложения. Эти данные зависят от окружения разработчика и не должны использоваться как production-конфигурация.

## Учетные данные

В коде есть тестовый вход администратора:


Эти данные предназначены только для локального учебного запуска проекта.
